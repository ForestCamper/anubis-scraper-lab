HTTP response header

The value never touches the HTML — it travels in a custom response header.

⚠ True response headers require a server. The included serve.py emits this header so you can test it for real.

Servers can return values in custom HTTP response headers (X-Access-Token), set cookies via Set-Cookie, or expose data in Link / ETag headers — none of which appear in the HTML body.

When you load this page through the bundled serve.py, the response carries:

HTTP/1.0 200 OK
Content-Type: text/html
X-Access-Token: FLAG{http-header-DADBF6EF}
how to extractInspect the HTTP response headers (e.g. fetch(url).then(r => r.headers.get('X-Access-Token')) or curl -I). The bundled server attaches the header to this page.
